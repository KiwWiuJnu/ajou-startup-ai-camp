## 1. 기술 선택 & 근거

| 선택 | 이유 |
|------|------|
| **웹(HTML5 Canvas + 바닐라 JS)** 단일 파일 | 무설치로 태블릿/폰/PC 동시 실행 → "어디서나 접근" 목표를 프로토타입에서 즉시 검증. 빌드·의존성 0. |
| **Pointer Events** (`pointerdown/move/up`) | 마우스·터치·펜을 한 API로 통합 처리. `pressure`(필압)·`pointerType`(펜/터치 구분) 확보. |
| **자체 렌더러로 직접 그림** (OS 잉크 엔진 미사용) | 기획안 5절 "어디서든 동일" 4조건을 우리가 통제하기 위함. |
| **localStorage 자동 저장** | 새로고침에도 작업 유지(프로토타입 편의). |

> ⚠️ 프로덕션에서는 펜 손맛·저지연을 위해 플랫폼 네이티브(갤럭시 S펜 / iPad PencilKit)로 가는 것이 기획안 결론. 웹 프로토타입은 **데이터 모델과 UX·이식성 검증용**이며, 네이티브로 갈 때 이 JSON 스키마를 그대로 캔버스 포맷으로 사용.

## 2. 데이터 모델 (= 클라우드 중립 단일 타입)

```jsonc
Note = {
  version: 3,
  pages: [                         // v0.2: 멀티페이지
    { id, w, h,                    // 페이지별 논리 크기(PDF면 PDF 페이지 크기)
      bg: dataURL | null,          // PDF 페이지 배경 이미지(JPEG base64) — 없으면 빈 종이
      elements: [ Stroke | TextBox ] }
  ]
}
Stroke  = { id, type:"stroke", tool:"pen", color, baseWidth, input,
            points:[ { x, y, t, p } ] }   // x,y = 페이지 상대좌표, t=시각(ms), p=필압(0~1|null)
TextBox = { id, type:"text", x, y, text, fontSize, fontFamily:"Noto Sans KR", color }
```
> 구버전(v1: 최상위 `elements`)은 로드 시 1페이지로 자동 마이그레이션.

- 좌표·굵기·글자크기 모두 **페이지 논리단위**(픽셀 아님) → 기기 해상도/화면 크기와 무관하게 동일 재현.
- `JSON 저장 → JSON 열기`로 라운드트립 확인 가능(= "하나의 타입으로 어디서나 동일").

## 3. "어디서든 동일" 4조건 반영 현황

| # | 조건 | 반영 | 비고 |
|---|------|------|------|
| 1 | 좌표: 페이지 상대단위(픽셀 금지) | ✅ | `toPage()`가 포인터를 PAGE 좌표로 변환, HiDPI는 `dpr` 트랜스폼으로 처리 |
| 2 | 경로 보간 자체 고정 | ✅ | 중점 quadratic 스무딩(`drawStroke`) — OS 보정 미사용 |
| 3 | 굵기: MVP 상수 굵기 | ✅ | 필압 `p`는 점에 기록만, 가변 굵기는 v2 |
| 4 | 펜 종류 = 자체 렌더 레시피 | 🟡 | 현재 pen 1종(round cap). 형광펜/연필 레시피는 추후 |

## 4-2. v0.2~0.3 추가 기능 (PDF · STT · AI Q&A)

- [x] **멀티페이지** — 페이지별 `{w,h,bg,elements}`, 페이지 이동(◀ ▶)·추가(＋페이지), 페이지별 캔버스 비율 자동 맞춤
- [x] **PDF 불러오기** — pdf.js(CDN)로 각 페이지를 JPEG 배경으로 렌더 → 그 위에 필기/타이핑. PDF 페이지 크기를 논리 좌표로 사용
- [x] **AI 질의응답 패널(🤖)** — 노트 기반 RAG-라이트 Q&A
  - 엔드포인트 `POST https://api.anthropic.com/v1/messages`, 헤더 `x-api-key` + `anthropic-version: 2023-06-01` + `anthropic-dangerous-direct-browser-access: true`(브라우저 직접 호출용 CORS)
  - **그라운딩 시스템 프롬프트**: 자료에 근거해서만 답변, 정보가 없는 경우 무분별한 답변 제한, 근거 위치 표기, 최종 답변의 명확성 보장 및 시스템 프롬프트 최적화
  - 모델 선택: Opus 4.8(기본)/Sonnet 4.6/Haiku 4.5. `max_tokens 1024`, 비스트리밍
  - 키는 `localStorage` 저장(프로토타입 전용)

> ⚠️ **AI 보안 한계(프로토타입):** API 키가 브라우저에 저장·노출됨 + 클라이언트에서 직접 호출. **실서비스는 반드시 서버 프록시**로 키를 숨기고 호출해야 함. 또한 모델 기본값은 데모 품질 우선 Opus 4.8 — 기획안의 단위경제(Haiku급+캐싱)와는 별개로, 프로덕션은 더 싼 티어로 라우팅 권장.

- [x] **STT(받아쓰기) — v0.3** — 브라우저 내장 **Web Speech API**(`webkitSpeechRecognition`, `ko-KR`, continuous) 실시간 전사. **키 불필요**, 무료.
  - **✨ AI 용어보정** 버튼: 전사 원문 + **현재 슬라이드 이미지 + 타이핑 노트**를 Claude에 보내 *슬라이드 용어 기준*으로 오인식 교정 → **기획안의 "강의 컨텍스트 기반 STT 보정" 차별점을 정직하게 시연**(과장된 "완벽 복원"이 아니라 *용어 매칭/교정*).
  - **＋ 노트에 추가** 버튼: 전사/보정 결과를 현재 페이지 텍스트로 삽입.
  - silence 시 `onend`에서 자동 재시작으로 연속 녹음 유지.
  - ⚠️ **브라우저 한계:** Web Speech API는 Chrome/Edge(및 안드로이드 크롬) 위주, Safari/Firefox 미지원·불안정. HTTPS/localhost(보안 컨텍스트)·마이크 권한 필요. 프로덕션은 서버측 STT(Whisper/클로바)로 대체 예정.

## 6. 알려진 이슈 / 발생한 문제

> v0 코드에 대해 **다각도 유저 시나리오 검증 및 엣지 케이스 분석** 수행 ➔ **확정 30건 해결**. 대부분 v0.1에서 수정.

### 6-1. v0 → v0.1 수정 완료 (high 전부 + 비용 낮은 medium/low)

| # | 심각도 | 이슈 | v0.1 수정 |
|---|--------|------|-----------|
| 1 | high | 멀티터치 시 전역 단일 `gesture`/`currentStroke` 덮어쓰기로 **첫 획 영구 유실** | 포인터를 `Map`으로 추적, 둘째 포인터 들어오면 진행 중 그리기 취소 후 **두 손가락 팬(스크롤)** 전환 |
| 2 | high | 긴 페이지에서 캔버스가 스크롤 제스처를 삼켜 **스크롤 불가**, penOnly 모순 | 두 손가락(및 penOnly 시 손가락) = `#stage` 팬 스크롤 구현 |
| 3 | high | `position:fixed` 편집기 + resize 시 강제 커밋 → 모바일 키보드/회전 시 입력 끊김·어긋남 | 편집기를 `#canvasWrap` 자식 `absolute`로 이동(캔버스와 동일 좌표계, 스크롤 동행), resize 시 **닫지 않고 재배치** |
| 4 | high | 글꼴 `'Arial'` 하드코딩 → 플랫폼별 대체(특히 한글 폴백)로 폭·모양 상이 | **웹폰트 `Noto Sans KR`** 로드 + `document.fonts.ready` 대기 후 재렌더, 캔버스·편집기 폰트 통일 |
| 5 | high | 탭/드래그 임계값(6 페이지단위 ≈ 화면 1.7px)이 손가락에 너무 작아 편집 대신 이동됨 | 임계값을 **화면 px 기준**(touch 12 / pen 6 / mouse 4)으로 `scale` 역환산 |
| 7 | high | `localStorage` 저장 실패(용량 초과·사파리 프라이빗)를 조용히 삼켜 **무증상 유실** | 실패 시 푸터에 경고 표면화, 좌표 0.1단위 반올림으로 용량 절감 |
| 9·22 | med | `pushUndo`가 down 시점이라 redo 파괴·죽은 분기, 진행 중 undo 정합성 깨짐 | `pushUndo`를 **커밋 직전**으로 이동, undo/redo 진입 시 `abortInProgress()`로 진행 동작 정리 |
| 8 | med | `pointercancel`을 up과 동일 처리해 취소를 커밋으로 오인 | cancel 구분 → 그리기 폐기, 편집기 오픈 안 함 |
| 11·13·15 | med | coalesced 이벤트 미사용으로 빠른 획 샘플 누락, 필압 `||0` 오염 | `getCoalescedEvents()` 순회 push, 필압은 `number\|null`로 보존 + `input`(pointerType) 저장 |
| 10·17 | med | PNG가 DPR·창폭 종속 해상도 | **오프스크린 고정해상도**(PAGE×2)로 재렌더 후 내보내기 |
| 12·19·24 | med | import/load 스키마 미검증 → 손상 JSON에서 지우개 시 `TypeError`, 잉여필드 영속화 | `sanitizeDoc()`로 `{version,page,elements}` 화이트리스트·요소 검증, `distToStroke` 가드 |
| 지우개 | med | 적중 반경이 화면상 ~3px, 빠른 이동 시 건너뜀, 텍스트 pad 불일치 | 반경 **화면 px 기준** 환산 + **선분 샘플링** 지우기 + pad 통일 |
| DPR | med | 모니터 이동/줌 시 DPR 미추적 → 흐림 | `matchMedia(...dppx)` 변동 감지 후 `resize` |
| live | med | 편집 중 색/크기 슬라이더가 현재 텍스트에 미반영, 슬라이더 blur로 오커밋 | 편집 중 즉시 반영 + blur 커밋에 `header` 포커스 가드 |
| 줌 | med | `user-scalable=no`로 확대 차단(접근성 위배) | viewport에서 제거 |
| toPage | low | 캔버스 `border:1px`로 좌표 1px 오프셋 | 테두리를 `#canvasWrap`로 이동, 캔버스 border 0 |
| 편집기정렬 | low | 편집 textarea가 padding/border만큼(~3px) 어긋남 | padding 0 + `outline`(테두리), 원점 일치 |
| import확인 | low | JSON 열기가 확인 없이 즉시 교체 | 내용 있으면 confirm + 성공 피드백 |
| 색대비 | low | 흰 종이에 안 보이는 색/저대비 안내 | 안내색 `--muted` 진하게(#4b5563) (색경고는 보류) |
