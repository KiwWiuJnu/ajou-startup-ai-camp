# 강의노트 AI: 사용자 여정 맵 (User Journey Map)

## 1. 개요 및 설계 원칙
*   **제품명:** CHRONOTE(크로노트) (한국 대학 강의 특화 통합 학습 노트)
*   **타겟 세그먼트:** 대학생
*   **제품 제1원칙:** **노트 우선, AI는 온디맨드 (Progressive Disclosure)**
    *   진입 마찰 최소화: 처음엔 일반 노트장처럼 동작하여 거부감 제거.
    *   단위경제 최적화: 마진을 해치는 AI 변동비(STT, RAG 토큰 비용)는 사용자가 기능을 켤 때만 발생하도록 유도.

## 3. 와이어프레임 및 디자인 핵심 가이드

### A. 메인 필기 캔버스 (단계 2)
*   **UX 핵심:** 필기감과 기능 정면 승부를 피하고 '데이터 입력 채널'로 포지셔닝. Apple PencilKit을 적극 활용하여 기본 잉크 성능 확보.
*   **UI 연출:** 화면 상단 툴바는 [펜, 형광펜, 지우개, 올가미, ★ 마커, 녹음]으로 극도로 단순화. 녹음 중 ★ 마커를 누르면 캔버스 해당 위치에 타임스탬프가 박힌 인덱스 아이콘이 생성됨.

### B. 복습 및 전공 스크립트 뷰 (단계 3)
*   **UX 핵심:** "자신감 있게 틀린 노트(환각)"의 독성을 방지하기 위한 안전장치 마련. LLM 후처리를 과신하지 않고 사용자가 검토할 수 있도록 유도.
*   **UI 연출:** 스크립트 타임라인에서 인공지능이 판단한 저신뢰 구간(Whisper 신뢰도 점수 기준 하위 구간)은 50% 불투명도의 회색 흐린 글씨로 처리하고, 마우스를 올리거나 터치하면 원본 오디오(3초 전부터)를 즉시 재생할 수 있는 작은 '듣기' 버튼 노출.

### C. 크로스소스 RAG 인터페이스 (단계 4)
*   **UX 핵심:** NotebookLM 등 무료 범용 툴이 구조적으로 불가능한 "필기 컨텍스트 결합 가치"의 증명.
*   **UI 연출:** 질문 창에 "중간고사 범위 내 ★ 표시된 부분 요약해줘" 입력 시, 답변 블록 내부의 핵심 문장마다 마우스 오버 시 하이라이트되는 하이퍼링크 인용 칩 구현. 칩 클릭 시 왼쪽 필기 화면이 해당 ★ 표시가 있는 노트 페이지와 당시 교수 발언 음성 스크립트 위치로 동시 스크롤 이동 처리.
